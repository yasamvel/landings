Архитектура Architecture

Mixture of Experts,
rethought from scratch

Hy3 uses a 295B MoE with 21B active parameters per token. Each token activates only 8 of 192 experts through a learned top-8 routing mechanism. This keeps inference fast while maintaining the knowledge capacity of a dense 295B model.

Top-8 Routing Top-8 Routing

Every input token is routed to the 8 most relevant experts among 192. The router is jointly trained end-to-end with the experts, minimizing load imbalance via auxiliary loss.

192 Эксперта 192 Experts

A large expert pool means each expert can specialize on narrower domains. Sparsity (8/192 active) keeps FLOPs per token comparable to a dense 21B model.

262K Контекст 262K Context

Extended context window of 262,144 tokens for processing entire codebases, research papers, and long conversations without truncation. RoPE scaling applied.

Эффективный Инференс Efficient Inference

With only 21B parameters active per forward pass, Hy3 delivers low latency and competitive throughput. Free tier available on OpenRouter for experimentation.
