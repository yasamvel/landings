Архитектура

Как устроен 3X-UI

Панель построена вокруг Xray-core и предоставляет веб-интерфейс для управления прокси-серверами, входящими и исходящими соединениями, а также каскадными цепочками.

01

Xray Core

Движок проксирования, поддерживающий VLESS, VMess, Trojan, Shadowsocks, WireGuard и SOCKS. Единый конфиг в JSON со строгой типизацией протоколов.

xray config.json
02

Веб-панель

Интерфейс на React с real-time статистикой, логами подключений, графиками трафика и управлением пользователями. REST API для автоматизации.

3x-ui web
03

База данных

SQLite для хранения конфигов, пользователей, статистики. Поддержка экспорта и бэкапов. Легковесность без внешних зависимостей.

/etc/3x-ui/
Каскад

Цепочки прокси-серверов

Каскадное проксирование позволяет выстраивать цепочки из нескольких серверов. Трафик проходит через промежуточные ноды последовательно, увеличивая анонимность и гибкость маршрутизации.

Исходящий (outbound)

Как это работает

В 3X-UI каскад настраивается через цепочку outbound-правил. Каждый последующий сервер получает трафик от предыдущего, шифруя его своим протоколом. Поддерживаются схемы:

  • → VLESS → Shadowsocks → Цель
  • → VMess → Trojan → SOCKS5 → Цель
  • → WireGuard → VLESS → Shadowsocks → Цель
Возможности

Управление нодами

3X-UI предоставляет полный контроль над каждым узлом инфраструктуры — от одного сервера до распределённого кластера.

Множественные ноды

Подключение неограниченного количества серверов через единую панель. Каждая нода со своим набором протоколов и пользователей.

Статистика в реальном времени

Графики трафика по каждому пользователю и ноде. Лог подключений с фильтрацией по дате, протоколу, IP.

Шаблоны конфигов

Быстрое развёртывание новых нод через предустановленные шаблоны. Поддержка JSON и TOML форматов.

API для автоматизации

RESTful API для интеграции с внешними системами: биллинг, мониторинг, оркестрация.

Двухфакторная аутентификация

TOTP и Telegram-уведомления для защиты панели от несанкционированного доступа.

Бэкапы и восстановление

Автоматические бэкапы конфигурации и базы данных с возможностью точечного восстановления.

Типы нод

Топология инфраструктуры

Каждая нода в 3X-UI может выступать в одной из трёх ролей. Комбинируя их, вы строите маршрут любой сложности.

активна

Входящая нода

Принимает подключения от клиентов. Первая точка входа в инфраструктуру. Настраивается с VLESS, VMess, Trojan.

inbound — порт 443
активна

Транзитная нода

Промежуточный узел в каскаде. Перенаправляет трафик без расшифровки. Увеличивает длину цепочки.

relay — цепочка
активна

Исходящая нода

Финальная точка перед целевым ресурсом. Именно её IP видит внешний сервер. Последний hop.

outbound — к цели
Конфигурация

Пример настройки

Базовая конфигурация Xray для каскадного проксирования с двумя промежуточными нодами.

// Каскад: клиент -> VLESS нода -> Shadowsocks -> цель
{
  "inbounds": [{
    "port": 443,
    "protocol": "vless",
    "settings": {
      "clients": [{ "id": "uuid-here" }]
    }
  }],
  "outbounds": [{
    "protocol": "shadowsocks",
    "settings": {
      "servers": [{
        "address": "node2.example.com",
        "port": 8388,
        "method": "chacha20-ietf-poly1305",
        "password": "password"
      }]
    }
  }, {
    "protocol": "freedom",
    "tag": "direct"
  }]
}
